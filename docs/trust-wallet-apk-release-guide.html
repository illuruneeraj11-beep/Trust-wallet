End-to-end journey

How the application works

Launch and authenticate

Email sign-in/account creation, connected-mode configuration validation, and native session persistence through SecureStore.

Create wallets

Create and name several testnet wallets, switch active wallet, inspect addresses, and archive only when ledger rules allow it.

Fund assets

Fund supported testnet assets through authenticated ledger functions with decimal-safe amounts and transaction journals.

Send by address

Validate address, network, amount, balance, fee, and idempotency before debit and recipient credit.

Receive and scan

Show wallet address/QR payload; use QR-only camera permission to populate address entry.

Review activity

Balances, activity, and notifications read from the ledger. Final two-device realtime observation remains required.

Browse live markets

Home, Markets, Search, Swap, and token detail use canonical assets and approximately 60-second fresh quote snapshots.

Explore other flows

Perps, Discover, browser, Buy, Swap, Settings, Preferences, Security, WalletConnect, and researched entry states remain browse/testnet-safe.

Final go/no-go

Two-phone acceptance checklist

  • Fresh install launches without Metro or a developer computer.
  • Account creation, confirmation, sign-in, sign-out, and session restore work.
  • Wallet create, rename, switch, archive guards, and address display work.
  • Funding updates only the selected user's wallet.
  • Sender decreases by amount plus fee exactly once.
  • Recipient increases by amount exactly once.
  • Repeat submit, slow network, and resume cannot duplicate transfer.
  • Wrong network, malformed address, overdraft, and archived recipient fail clearly.
  • Recipient activity updates realtime or immediately after refresh.
  • QR permission, scan, flashlight, and denial recovery work.
  • Copy/paste, numeric keyboard, keyboard avoidance, and back work.
  • Safe areas and navigation fit small and tall Android screens.
  • Home, Markets, Search, Swap, token detail, Settings, and secondary routes do not dead-end.
  • Offline, stale, timeout, and reconnect states are truthful.
  • No secret phrase, private key, real payment, or real signing prompt appears.
  • Name, icon, About, support, privacy, and terms identify the publisher accurately.
Release rule: one failed security, identity, ledger, install, or crash item is a no-go. Fix it, rebuild, and repeat both-device testing from a fresh install.

Distribution

How to share after it passes

Android

Share the EAS internal URL only with intended testers. Tell them the exact publisher/build identity and that it is an unofficial testnet prototype. Restrict Expo project access if the link must remain private.

iPhone 17 and iOS

An APK cannot run on iPhone. iOS needs a separate signed build, Apple Developer membership, TestFlight/ad-hoc provisioning, iOS physical testing, and its own distribution path.

Runtime prices: Exa is for research, not price delivery. Current production already returns fresh main-asset quotes without Exa. A server CMC key is optional for history/faster streaming only after proxy abuse controls are added.
